# MC Nylon Defect Analysis Skill

当用户描述实验失败、成型异常、外观缺陷、性能波动、分散问题或聚合问题时，使用这个
skill。

## 解决什么问题

- 把“现象描述”拆成可排查的工艺和材料节点
- 建立“症状 -> 可能原因 -> 验证方法 -> 修正动作”的诊断框架
- 区分高概率原因、低证据推测和必须回看实验记录的部分
- 帮用户决定下一步先验证什么，而不是盲目重做全部实验

## 什么时候触发

- “这批材料有气泡/孔洞”
- “样品发白、黄变、脆裂”
- “加了填料以后分散不良、团聚、沉降”
- “聚合不完全，无法成型”
- “性能波动很大，不知道是哪里出了问题”

## 建议输入

1. 具体症状
   - 外观、断口、色泽、尺寸、流动性、成型状态、性能异常
2. 工艺记录
   - 脱水、真空、加入顺序、浇铸、模具、聚合、保温
3. 材料与配比
   - monomer、catalyst、activator、TDI、NaOH、glass fiber、silica 等
4. 对照信息
   - 与上一次成功批次相比改了什么
5. 证据形式
   - 图片、视频、记录表、测试结果、主观观察

## 默认工作方式

1. 先把症状标准化。
2. 按以下层次排查：
   - 原料与含水
   - 催化/活化与配比
   - 真空与脱水
   - 温度与时间窗口
   - 浇铸与模具
   - 填料加入与分散
   - 后处理与测试条件
3. 只给出与当前症状相关的原因，不泛化成大而空的列表。
4. 对每个可能原因都配一个可执行验证动作。
5. 优先建议低成本、能区分原因的验证顺序。

## 默认输出格式

```text
问题描述整理
- ...

症状诊断表
- 症状：
  可能原因：
  验证方法：
  修正动作：

优先排查顺序
- ...

需要回看或补充的实验记录
- ...

当前结论可信度
- ...
```

## 不能做什么

- 不在没有记录时假定配比、真空度、温度已经正确
- 不把缺陷直接归因到单一因素
- 不编造机理、性能损失幅度或标准答案
- 不给“万能修复建议”
- 不跳过验证环节直接给确定结论

## 不确定时如何处理

- 说明“当前只能给高概率假设，需靠记录或对照实验验证”
- 若证据不足，先要求补充症状照片、工艺记录或对照批次信息
- 若多个原因都可能成立，按验证成本和区分度排序
- 若缺陷跨越多个环节，优先回到最近发生变化的变量

## 输出风格要求

- 中文为主，保留必要英文术语
- 先给诊断表，再给短解释
- 强调“验证动作”，避免只给概念判断
- 任何结论都要回扣到具体记录或待验证证据
